# System Boundary Interface Documentation

This document describes the system's external invocation interfaces, including CLI commands, API endpoints, configuration parameters, and other boundary mechanisms.

## Command Line Interface (CLI)

### kimi

**Description**: Primary CLI entry with multi-mode support (shell/print/ACP/wire), session management, config loading, and OAuth integration

**Source File**: `src/kimi_cli/cli/__init__.py`

**Arguments**:

- `prompt` (string): optional - Direct prompt input for immediate execution
- `session_id` (string): optional - Specific session ID to resume
- `work_dir` (string): optional - Working directory for the session

**Options**:

- `--version, -v`(boolean): optional - Show version information
- `--verbose`(boolean): optional - Enable verbose logging
- `--debug`(boolean): optional - Enable debug mode
- `--continue, -c`(boolean): optional - Continue last session for directory
- `--print`(boolean): optional - Print mode for non-interactive output
- `--acp`(boolean): optional - ACP protocol mode
- `--wire`(boolean): optional - Wire protocol mode (JSON-RPC over stdio)
- `--thinking`(boolean): optional - Enable thinking mode
- `--yolo`(boolean): optional - Auto-approve all tool executions
- `--model`(string): optional - Specify model name to use
- `--config`(string): optional - Configuration string override

**Usage Examples**:

```bash
kimi --prompt 'List all Python files'
```

```bash
kimi --session abc123 --prompt 'Show me the main function'
```

```bash
kimi --continue --prompt 'Refactor this code'
```

```bash
kimi login
```

```bash
kimi logout
```

```bash
kimi --wire
```

**Example Code**:

```

  createSessionRequest: { work_dir: '/path/to/project', create_dir: false }

  jsonrpc: '2.0',
  id: 'prompt-1',
  method: 'prompt',
  params: { user_input: 'Explain this code' }
}));
```

**Best Practices**:

- Always include session token in WebSocket query parameter for authentication
- Handle JSON-RPC 2.0 message format correctly with proper id tracking
- Implement reconnection logic for WebSocket disconnections
- Check busy state before sending new prompts to avoid rejection

### Wire Protocol Client

JSON-RPC based wire protocol for programmatic integration over stdio

**Example Code**:

```
import asyncio
import json

async def wire_client():
    proc = await asyncio.create_subprocess_exec(
        'kimi', '--wire',
        stdin=asyncio.subprocess.PIPE,
        stdout=asyncio.subprocess.PIPE
    )
    
    # Initialize
    init_msg = {
        'jsonrpc': '2.0',
        'id': 'init-1',
        'method': 'initialize',
        'params': {
            'protocol_version': 1,
            'client': {'name': 'my-client', 'version': '1.0'}
        }
    }
    proc.stdin.write(json.dumps(init_msg).encode() + b'\n')
    await proc.stdin.drain()
    
    # Read response
    line = await proc.stdout.readline()
    response = json.loads(line.decode())
    print('Initialized:', response)
    
    # Send prompt
    prompt_msg = {
        'jsonrpc': '2.0',
        'id': 'prompt-1',
        'method': 'prompt',
        'params': {'user_input': 'Hello'}
    }
    proc.stdin.write(json.dumps(prompt_msg).encode() + b'\n')
    await proc.stdin.drain()
```

**Best Practices**:

- Always send 'initialize' message before any other requests
- Use JSON-RPC 2.0 format with unique message IDs
- Handle message delimiting with newline characters
- Implement proper error handling for protocol version mismatches
- Buffer size limit is 100MB - do not exceed
- Current wire protocol version is 1.3

### Security Best Practices

Critical security considerations when integrating with the Kimi CLI system

**Example Code**:

```
# Security configuration example
# config.toml

[security]
restrict_sensitive_apis = true  # Disable open-in and deep file access
lan_only = true  # Restrict to LAN access only
enforce_origin = true  # Enable origin validation

# Path filtering for sensitive files
# Avoid exposing: .ssh/, .aws/, .gnupg/, credentials files

# Use environment variables for API keys
[providers.openai]
api_key = "${OPENAI_API_KEY}"  # Reference env var instead of hardcoding
```

**Best Practices**:

- Always enable restrict_sensitive_apis in production deployments
- Use LAN-only mode for local development security
- Never store API keys in config files - use environment variables
- Validate session tokens on all API requests in public mode
- Implement path traversal protection for file operations
- Sanitize filenames during uploads to prevent directory traversal
- Monitor for symlink attacks when accessing work directory files
- Be aware that yolo mode auto-approves all shell commands - use with extreme caution
