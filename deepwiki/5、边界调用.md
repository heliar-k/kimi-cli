# System Boundary Interface Documentation

This document describes the system's external invocation interfaces, including CLI commands, API endpoints, configuration parameters, and other boundary mechanisms.

## Command Line Interface (CLI)

### kimi

**Description**: Kimi CLI 主入口命令，提供交互式会话、非交互式处理、会话恢复等多种运行模式

**Source File**: `src/kimi_cli/cli/__init__.py`

**Arguments**:

- `prompt` (string): optional - 直接传入的提示文本，用于非交互式模式
- `session_id` (string): optional - 要恢复的会话ID
- `config_string` (string): optional - 配置字符串，直接传入配置内容
- `config_file` (Path): optional - 配置文件路径

**Options**:

- `--version`(boolean): optional - 显示版本信息 (default: `false`)
- `--verbose`(boolean): optional - 启用详细输出模式 (default: `false`)
- `--debug`(boolean): optional - 启用调试模式 (default: `false`)
- `--work-dir`(Path): optional - 本地工作目录路径
- `--continue`(boolean): optional - 继续上一个会话 (default: `false`)
- `--model-name`(string): optional - 指定模型名称
- `--thinking`(boolean): optional - 启用思考模式
- `--yolo`(boolean): optional - 启用YOLO模式（自动确认） (default: `false`)
- `--print`(boolean): optional - 启用打印模式（非交互式） (default: `false`)
- `--acp`(boolean): optional - 启用ACP模式 (default: `false`)
- `--wire`(boolean): optional - 启用Wire模式 (default: `false`)

**Usage Examples**:

```bash
kimi --work-dir /path/to/project
```

```bash
kimi --print --prompt "Analyze the codebase"
```

```bash
kimi --session <session-id>
```

```bash
kimi web --port 8080
```

## API Interfaces

### GET GET /api/sessions/

**Description**: 列出所有会话，支持分页、搜索和归档过滤

**Source File**: `src/kimi_cli/web/api/sessions.py`

**Request Format**: Query parameters: limit (int, default=100), offset (int, default=0), q (string, optional), archived (boolean, optional)

**Response Format**: JSON array of Session objects

**Authentication**: Session token authentication

### POST POST /api/sessions/

**Description**: 创建新的会话实例

**Source File**: `src/kimi_cli/web/api/sessions.py`

**Request Format**: JSON: CreateSessionRequest { work_dir?: string, title?: string }

**Response Format**: JSON: Session object

**Authentication**: Session token authentication

### WEBSOCKET WebSocket /api/sessions/{session_id}/stream

**Description**: WebSocket流式接口，用于实时会话消息推送

**Source File**: `src/kimi_cli/web/api/sessions.py`

**Request Format**: WebSocket connection with session_id and token query parameter

**Response Format**: JSON streaming messages

**Authentication**: Query parameter token for WebSocket authentication

### GET GET /api/sessions/{session_id}/files/{path:path}

**Description**: 获取会话中的文件内容

**Source File**: `src/kimi_cli/web/api/sessions.py`

**Request Format**: Path parameters: session_id (UUID), path (string)

**Response Format**: File content as Response

**Authentication**: Session token authentication

### POST POST /api/sessions/{session_id}/files

**Description**: 上传文件到指定会话

**Source File**: `src/kimi_cli/web/api/sessions.py`

**Request Format**: Multipart form data with UploadFile

**Response Format**: JSON: UploadSessionFileResponse { file_path: string, size: int }

**Authentication**: Session token authentication

### GET GET /api/config/

**Description**: 获取全局配置快照

**Source File**: `src/kimi_cli/web/api/config.py`

**Request Format**: No request body

**Response Format**: JSON: GlobalConfig { default_model: string, default_thinking: boolean, models: ConfigModel[] }

**Authentication**: Session token authentication

### PATCH PATCH /api/config/

**Description**: 更新全局配置，可选择重启运行中的会话

**Source File**: `src/kimi_cli/web/api/config.py`

**Request Format**: JSON: UpdateGlobalConfigRequest { default_model?: string, default_thinking?: boolean, restart_running_sessions?: boolean, force_restart_busy_sessions?: boolean }

**Response Format**: JSON: UpdateGlobalConfigResponse { updated_sessions: string[] }

**Authentication**: Session token authentication

## Integration Suggestions

### Python CLI集成

通过Python API直接创建和管理KimiCLI会话，适合在自动化脚本或应用中集成Kimi功能

**Example Code**:

```
from kimi_cli.app import KimiCLI
from kimi_cli.session import Session
from kaos.path import KaosPath

# 创建会话
session = await Session.create(KaosPath.cwd())
cli = await KimiCLI.create(session, model_name='kimi-k2-5')

# 运行交互
result = await cli.run('Explain this function')
```

**Best Practices**:

- 使用 KimiCLI.create() 工厂方法进行正确初始化
- 处理 LLMNotSet、LLMNotSupported、MaxStepsReached 等异常
- 实现正确的清理逻辑，处理 asyncio.CancelledError
- 建议在异步上下文中使用 async/await 模式

### React Web应用集成

在React应用中使用提供的Hooks和API客户端进行会话管理和配置控制

**Example Code**:

```
import { useSessions } from '@/hooks/useSessions';
import { useGlobalConfig } from '@/hooks/useGlobalConfig';

function MyComponent() {
  const { sessions, createSession, selectSession } = useSessions();
  const { config, update } = useGlobalConfig();
  
  // 创建新会话
  await createSession('/path/to/workdir');
  
  // 更新全局配置
  await update({ defaultModel: 'kimi-k2-5' });
}
```

**Best Practices**:

- 使用 useSessions 和 useGlobalConfig Hooks 进行状态管理
- 实现 WebSocket 重连逻辑以保持会话流连接
- 处理认证令牌刷新和过期场景
- 在组件卸载时正确清理 WebSocket 连接

### WebSocket实时流集成

**Example Code**:

```

ws.onmessage = (event) => {

  console.log('Received:', message);
};

ws.onerror = (error) => {
  console.error('WebSocket error:', error);
};
```

**Best Practices**:

- 实现心跳检测和自动重连机制

- 使用消息队列缓冲断线期间的消息
- 正确处理二进制和文本消息格式

### 自定义工具扩展

通过扩展 CallableTool2 接口创建自定义工具，集成到KimiCLI工具集中

**Example Code**:

```
from kaos.tools import CallableTool2, ToolOk, ToolError
from pydantic import BaseModel

class MyToolParams(BaseModel):
    input_text: str
    max_length: int = 100

class MyTool(CallableTool2[MyToolParams]):
    async def __call__(self, params: MyToolParams):
        try:
            result = process_text(params.input_text, params.max_length)
            return ToolOk(result)
        except Exception as e:
            return ToolError(str(e))
```

**Best Practices**:

- 使用 Pydantic 参数模型定义工具参数
- 返回 ToolOk 或 ToolError 确保正确的结果处理
- 使用 SimpleToolset 或 KimiToolset 注册工具
- 实现适当的错误处理和日志记录

### LLM Provider实现

实现 ChatProvider 接口以支持自定义LLM提供商集成

**Example Code**:

```
from kosong import ChatProvider, StreamedMessagePart

class MyChatProvider(ChatProvider):
    async def stream(
        self,
        messages: list[Message],
        **kwargs
    ) -> AsyncIterator[StreamedMessagePart]:
        # 实现流式响应
        async for chunk in self._call_llm(messages):
            yield StreamedMessagePart(content=chunk)
    
    def with_thinking(self) -> 'MyChatProvider':
        # 支持思考模式
        return self._with_thinking_enabled()
```

**Best Practices**:

- 支持通过 AsyncIterator 的流式响应
- 实现 with_thinking() 方法支持思考模式
- 正确处理 token 使用统计
- 实现适当的重试和错误处理机制
